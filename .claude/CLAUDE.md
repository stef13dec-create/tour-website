# Claude Code — Skills Reference

Invoke any skill by typing `/<skill-name>` in the prompt.

## Workflow & Process
| Skill | When to use |
|-------|-------------|
| `/brainstorming` | Before any new feature — design first, code second |
| `/writing-plans` | Turn a spec into a step-by-step implementation plan |
| `/executing-plans` | Execute a written plan in a separate session |
| `/subagent-driven-development` | Execute a plan using fresh subagents per task (same session) |
| `/finishing-a-development-branch` | When done — merge, PR, or discard options |
| `/using-git-worktrees` | Isolate feature work in its own worktree |

## Quality & Review
| Skill | When to use |
|-------|-------------|
| `/verification-before-completion` | Before claiming anything is done or committing |
| `/requesting-code-review` | After completing a feature or before merging |
| `/receiving-code-review` | Before implementing review feedback |
| `/systematic-debugging` | When hitting a bug — root cause before any fix |
| `/test-driven-development` | When implementing any feature or bugfix |

## Advanced
| Skill | When to use |
|-------|-------------|
| `/dispatching-parallel-agents` | Split 2+ independent tasks across parallel agents |
| `/using-superpowers` | Meta-skill — how to find and use skills effectively |
| `/writing-skills` | Create or edit a skill |
| `/skill-creator` | Scaffold a new skill interactively |
